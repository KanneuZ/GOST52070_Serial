# GOST52070_Serial

Программный модуль формирования и пакетной отправки управляющих сигналов для взаимодействия с микроконтроллером по стандарту ГОСТ Р 52070-2003 (ГОСТ МКО).

## Описание

Модуль реализует протокол мультиплексного канала обмена (аналог MIL-STD-1553B) для работы через последовательный интерфейс в ОС Windows.

**Основные возможности:**
- Кодирование и декодирование Манчестер II
- Формирование командных, ответных и данных слов
- Расчет бита четности
- Отправка и прием через COM-порт (WinAPI)
- Команды управления (таблица 1 ГОСТ)
- Полный цикл обмена (команда → данные → ответ)
- Режимы контроллера шины (КШ) и оконечного устройства (ОУ)

## Требования

- ОС Windows (7/10/11)
- MinGW или Visual Studio с поддержкой C++17
- Виртуальный COM-порт для тестирования (например, com0com)

## Сборка

make

Очистка объектных файлов:

make clean

## Запуск

./GOST52070_Serial.exe COM3 01 1

**Параметры:**
- COM3 — имя последовательного порта
- 01 — адрес оконечного устройства (hex, 0-31)
- 1 — режим работы (1=КШ, 0=ОУ)

Пример запуска в режиме ОУ на порту COM1 с адресом 5:

./GOST52070_Serial.exe COM1 05 0

## Команды

После запуска программа предоставляет интерактивное меню:

1 - Отправить команду (КШ)
2 - Отправить данные
3 - Отправить ответное слово (ОУ)
4 - КУ «Передать ОС»
5 - КУ «Сброс»
6 - КУ «Блокировать передатчик»
7 - КУ «Разблокировать передатчик»
8 - КУ «Начать самоконтроль»
9 - Принять слово
10 - Полный цикл обмена (КШ)
0 - Выход

## Форматы слов

Командное слово (КС):
- Разряды 1-3: Синхросигнал (3 бита)
- Разряды 4-8: Адрес ОУ (5 бит)
- Разряд 9: Прием/Передача (1 бит)
- Разряды 10-14: Подадрес (5 бит)
- Разряды 15-19: Число СД (5 бит)
- Разряд 20: Четность (1 бит)

Слово данных (СД):
- Разряды 1-3: Синхросигнал (3 бита)
- Разряды 4-19: Данные (16 бит)
- Разряд 20: Четность (1 бит)

Ответное слово (ОС):
- Разряды 1-3: Синхросигнал (3 бита)
- Разряды 4-8: Адрес ОУ (5 бит)
- Разряд 9: Ошибка в сообщении (1 бит)
- Разряды 10-19: Признаки состояния (10 бит)
- Разряд 20: Четность (1 бит)

## Тестирование

Для тестирования рекомендуется использовать виртуальные COM-порты:
1. Установите эмулятор (например, com0com или Virtual Serial Port Emulator)
2. Создайте пару портов (например, COM1 и COM2)
3. Запустите модуль на одном порту
4. Используйте терминальную программу (TeraTerm, Putty) на втором порту для мониторинга
